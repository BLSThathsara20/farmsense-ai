1. Recommendation

Use PostgreSQL 16

Already chosen in your dissertation stack (§5.5). Strong JSONB support for ranked recommendation audit, ACID for user data, free, and works with FastAPI + SQLAlchemy.

ORM SQLAlchemy 2 + Alembic

Migrations track schema changes for viva evidence. Alternative for speed: run database/init.sql once, then add Alembic when tables change.

v1 scope: One farm_profile per user is enough for the dissertation prototype. Multi-farm support is a one-to-many relationship already — add UI later without schema changes.
Do not store in PostgreSQL: raw ML model files (.pkl, .h5) — keep those in ml-models/artifacts/. The DB stores outputs and metadata only.

2. Design principles

  • Lineage: account → farm → soil reading → recommendation run (audit for FR6).
  • Privacy (NFR2): district_crop_aggregate stores counts only — no user IDs in collective tables.
  • Reference vs transactional: crop_reference and market_snapshot are shared; user tables are isolated per account.
  • JSONB audit: recommendation_run.ranked_output mirrors the React API shape so pytest and UI stay aligned.
  • UK focus: district_code can use ONS codes later; free-text district name is fine for prototype.

7. Key queries

Latest recommendation for a farm

SELECT ranked_output, top_crop, created_at FROM recommendation_run WHERE farm_profile_id = $1 ORDER BY created_at DESC LIMIT 1;

District popularity (Community page)

SELECT c.display_name AS crop, a.plan_count AS farmers, ROUND(100.0 * a.plan_count / SUM(a.plan_count) OVER (), 1) AS percentage FROM district_crop_aggregate a JOIN crop_reference c ON c.id = a.crop_id WHERE a.district_code = $1 AND a.season_year = $2 AND a.week_number = $3 ORDER BY a.plan_count DESC;

Oversupply check (TC06 logic)

SELECT district_code, crop_id, plan_count AS current_count, $threshold AS threshold_count, (plan_count >= $threshold) AS warning FROM district_crop_aggregate WHERE district_code = $1 AND crop_id = $2;

8. Docker & local setup

Start PostgreSQL locally (one command):

cd farmsense-ai docker run --name farmsense-db -e POSTGRES_PASSWORD=farmsense \ -e POSTGRES_DB=farmsense -p 5432:5432 -d postgres:16 # Apply schema docker exec -i farmsense-db psql -U postgres -d farmsense < database/init.sql

Backend .env (add when wiring SQLAlchemy):

DATABASE_URL=postgresql+psycopg://postgres:farmsense@localhost:5432/farmsense JWT_SECRET=change-me-in-production

Suggested Python packages: sqlalchemy>=2.0, psycopg[binary], alembic, passlib[bcrypt], python-jose.
