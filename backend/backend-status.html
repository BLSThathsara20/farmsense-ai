FarmSense AI — Backend live reference

Backend architecture & performance

Layered FastAPI service connected to PostgreSQL 16, React frontend, and ML artefacts. Optimised for fast JSON responses, safe retries, and future scale.

Performance techniques (production patterns)

ORJSON responses

FastAPI default response class — faster JSON serialisation than stdlib.

ML singleton

Random Forest loaded at startup — no disk I/O per request.

Connection pool

SQLAlchemy pool_pre_ping, size 10, overflow 20.

TTL response cache

GET /dashboard, /market, /community cached 60s in-process.

GZip middleware

Compresses responses >500 bytes.

X-Response-Time-Ms

Every response includes server timing for profiling.

Future upgrades: Redis for distributed cache + idempotency; --workers N behind nginx; read replicas for market snapshots; async SQLAlchemy + asyncpg under load.

Security

  • JWT Bearer tokens (HS256) — frontend stores token in Zustand persist
  • Bcrypt password hashing via Passlib
  • CORS restricted to Vite dev origins
  • Rate limiting via SlowAPI (120/min default)
  • Security headers: X-Content-Type-Options, X-Frame-Options, Referrer-Policy
  • Structured error JSON — no stack traces to clients
  • District aggregates store counts only (NFR2)

Idempotency (safe POST retries)

Send header Idempotency-Key: <uuid> on POST /soil. Same key + same body replays stored response; mismatched body returns 409. Table: idempotency_record.

Run locally

cd farmsense-ai
docker compose up -d --build

# Backend (without Docker)
cd backend && cp .env.example .env
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/uvicorn app.main:app --reload --port 8000

# Frontend (live API)
cd frontend && cp .env.example .env
npm run dev

# Tests
cd backend && .venv/bin/pytest tests/ -v
